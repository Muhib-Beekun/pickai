# Data Lineage

## Sources

1. Public warehouse dataset: https://data.mendeley.com/datasets/pf2w725pw3/1
2. Local fallback fixture generation when direct download is blocked.

## Mendeley handling

- `scripts/download_mendeley.py` attempts direct dataset download/extract.
- If unavailable, the script generates schema-aligned CSV fallback files in `data/mendeley/` and logs fallback behavior.
- CI and deterministic tests use `data/fixtures/mendeley_sample.csv` (<=200 rows).

## Synthetic training data

- Generated by `scripts/generate_synthetic_jsonl.py`.
- Each row is produced from sampled order lines and deterministic optimizer output, not hallucinated labels.
- Full set lives in `data/synthetic/synthetic_nl_parse.jsonl` (gitignored).
- Committed sample lives in `data/synthetic/sample.jsonl`.

## Refined synthetic data

- `scripts/agentic_nl_refine.py` runs writer/validator cycles on held-out rows.
- High-scoring rows are written to `data/synthetic/refined_nl_parse.jsonl`.
- Per-cycle audit logs are written to `logs/agentic_refine.jsonl`.

## Privacy and licensing

- No customer WMS data is committed.
- Public release remains MIT with upstream attribution retained in `NOTICE.md`.
